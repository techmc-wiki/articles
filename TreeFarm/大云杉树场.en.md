> Welcome to the primary source of wood for mid-to-late game engineering. Here we will gradually guide you through designing the most efficient log-producing tree farm in Minecraft.

## Introduction to Large Spruce Tree Farms

### Large Spruce Growth Mechanics and Architecture Design

The growth detection range for large spruce is: **centered on the northwest corner sapling, 3x3 at sapling layer, 5x5 above**

The maximum height of a large spruce is 28 blocks (the northwest corner will have one extra, making it 29) ~~generally speaking, no one would build a large spruce tree farm without processing full height~~

Simply put, we need a double recursion to process the trunk logs, then simply push away the leaves with honey-slime walls.

### Vertical Signal Transmission

Just use stick bud to pull it up. For trunk and side wall activation, you can consider directional Redstone Dust or wall power.

### Processing

After pushing out from the core, split left and right, then reassemble normally.

At this point, you can already independently design a functional large spruce tree farm. But if nothing goes wrong, the efficiency will only be around 100k, which is unsatisfactory. Next, we'll discuss how to make a faster large spruce tree farm. For processing timing, I believe you have gained sufficient understanding in previous studies, so here we'll only discuss how to maximize pre-bonemealing.

## [!ADVANCED]High-Speed Large Spruce Part 1: Corner Down-Suction

### The Origin of Corner Down-Suction

In 2016, Laoxian released his 12gt large spruce tree farm, using an upward-pushing architecture, breaking through the 300k mark for the first time. However, as time passed, Laoxian's timing no longer worked in 1.12, and players gradually discovered better architecture and timing designs.

On February 12, 2020, gpw released sprucemacy v1, using Observer-powered tree root detection and corner down-suction, officially opening the era of modern high-speed large spruce tree farms, becoming the design standard for large spruce for quite some time, until April 15, 2021, when gpw released sprucemacy v2, bringing corner side-suction, which we'll explain a bit later. Now, let's go back to 2020.

### The Concept of Corner Down-Suction

If you tried to build a large spruce tree farm in 8.1, you probably noticed that it's quite difficult to add dispensers to this thing. Without modifying the base architecture, even using push limit detection, we can only have two dispensers.

*Since we're still in 2020, we don't have push limit detection/tree power detection yet, so let's ignore them for now.* We need to choose one of the four positions where dispensers can be placed to put the powered tree root detection. Obviously, if we want to increase to three dispensers, we inevitably face a problem: no matter what, we cannot process the corner log through side-suction.

The solution is obvious: just down-suck it. But because the dirt return speed of down-suction is slower, we need to design a separate planting timing.

gpw's solution was to have the player plant the corner sapling at 8/10gt (in 1.12, player right-click placement runs under a simple 4gt interval), and the rest can be planted normally.

For player movement, gpw used minecarts.

### Limitations of Corner Down-Suction

In 1.14, Mojang modified the player right-click placement behavior. If due to some issues, the player first places the sapling on the side closer to the player, then the sapling farther from the player cannot be placed. Currently, without modifying the planting timing, the only solution is llama boats ~~(but in 1.21, because Mojang changed the behavior of mobs on boats again, llama boats also broke)~~

Corner down-suction in 1.12, even with 3gt dirt return, cannot guarantee planting the corner sapling before 4gt (the reason is the right-click placement logic mentioned above), so it cannot reach the ultimate efficiency.

## [!ADVANCED]High-Speed Large Spruce Part 2: Corner Side-Suction (Corner Retraction)

### The Origin of Corner Side-Suction

On April 15, 2021, gpw released sprucemacy v2, bringing us corner side-suction for the first time. On February 8, 2022, floppy's 12gt Spruce v2 brought corner side-suction timing that completes bonemealed sapling planting in 3gt. In August 2024, Qontrol developed a corner side-suction base with practical significance that resets in 6gt.

> Actually, not long after 12gt corner side-suction came out, Xinghe casually made an all-dust 6gt, ~~but it didn't look usable at all~~

~~Because Mojang changed llama boats in 1.21, the significance of corner side-suction was also destroyed~~

### The Concept of Corner Side-Suction

If you want to plant the corner sapling first, you must side-suck it. The side of the corner trunk has two dispensers and two logs. If we can somehow remove one log from the side, we can insert a side-suction Piston. This is corner side-suction.

The current implementation is to down-suck one of the logs, and the other side can be processed with conventional parallel side-suction.

For corner side-suction, obviously we need to use double recursion. For 16gt non-auto-clicker large spruce, just use normal double recursion; for 12gt, we use double recursion with secondary Pistons pushing in from the side; for 6gt, we use a design similar to the "standard flow" of retract-rotate-push.

Generally, player movement uses llama boats with corner-priority planting timing; or Piglin boats, planting one on the parallel side-suction side first, then the corner, then the second one on the parallel side-suction side, and finally planting the down-suction position.

> We will describe the specific origin of Piglin boats in the next section.

### Too Difficult to Build

If you still remember the 8gt large spruce, you'll know that corner side-suction didn't appear in that tree farm. The reason is actually very simple: neither lintex nor I could think of a good enough wiring method to handle 8gt corner side-suction. So we turned to corner down-suction and changed the planting timing (after all, no one would use a real person without auto-clicker for 8gt large spruce
