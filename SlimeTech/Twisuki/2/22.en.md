# #22 Semi-Dynamic Engines and Extensions

---

In this section, we explore two special flying machine engines, structures that rely on other independent engines to pass motion forward.

## Semi-Dynamic Engines

A `semi-dynamic engine` is a counting engine that moves one block when it receives an external signal. As shown in the figure:

![img.gif](img/2.2.1-bud.gif)

The above semi-dynamic engine structure is simple and very commonly used, but it cannot work properly with the now-common 10gt standard engine. Therefore, we explore how to make a flying machine go from `cannot move`, to `always moving`, to `move once`.

As mentioned in the previous section, the essential structure of a flying machine is:

- Two pistons facing each other, at least one being a sticky piston
- Two observers, each activating one piston
- Slime Blocks for connection

The two observers are the key to achieving `always moving`, as they perfectly provide the timing required for continuous motion.

![img.png](img/2.2.1-10gt.png)

Analyzing the 10gt flying machine, we can see that the next movement is provided by the left observer arriving in position and providing a signal again, causing the entire flying machine to enter the next cycle. So to make the flying machine move only once, we just need to prevent the observer from providing the signal to the piston when it arrives in position.

As shown in the figure, we mount the left observer to the right structure. This way, when starting, the observer can provide a signal to the piston, but when it arrives in position, the observer and piston are separated exactly, preventing the next cycle from starting.

![img.png](img/2.2.1-10gt半动.png)

![img.png](img/2.2.1-10gt半动原理.png)

_Note: the left piston in the figure cannot be a sticky piston. Due to signal length issues, if replaced with a sticky piston, it will cause the structure to oscillate._

We simplify the above semi-dynamic engine structure and add a Note Block as a switch:

![img.png](img/2.2.1-半动机.png)

From this we can summarize the essential structure of a simple semi-dynamic engine:

- Two pistons facing each other, the one facing the direction of travel is a regular piston, the one facing away from the direction of travel is a sticky piston
- Two observers that can activate both pistons, but mounted on the same side
- Slime Blocks for connection

In fact, various tunnel bores and unidirectional bulldozers currently in use are all large semi-dynamic engines. They receive signals from the player, move forward one block, reset, and wait for the next signal to arrive.

![img.png](img/2.2.1-推土机.png)

_The figure shows my unidirectional bulldozer and the extracted semi-dynamic switch._

## Extensions

We explained flying machine mounting and extensions in `1.3.3`, so here we will no longer distinguish between mounting and extensions.

### a) Motion Extensions

The simplest extension structure is a forward-facing regular piston and a backward-facing sticky piston. Using observers and pistons, motion is passed forward. This will not be explained further here.

### b) Signal Extensions

Combining the above semi-dynamic engine, we use a flying machine to provide periodic signals to the semi-dynamic engine, and the semi-dynamic engine can move periodically, synchronized with the flying machine.

![img.png](img/2.2.2-10gt飞行器与半动机.png)

In the figure, the upper structure is a regular 10gt flying machine with an additional observer mounted, and the lower structure is a semi-dynamic engine. They can move synchronously.

![img.gif](img/2.2.2-10gt飞行器与半动机运行.gif)

_Here you might ask: the characteristic of a semi-dynamic engine is to move only once, but we use a flying machine to connect to the semi-dynamic engine, doesn't that make the semi-dynamic engine "useless"? Why not just make another flying machine directly?_

_In fact, a semi-dynamic engine is a very simple extension structure. On one hand, the main flying machine may reach the push limit and cannot pass motion forward through observers and pistons, in which case a relatively lighter semi-dynamic engine can be used; on the other hand, a semi-dynamic engine is not `move only once`, but rather `stop after moving`, meaning that if the main flying machine stops for some reason, the semi-dynamic engine as an extension must stop immediately. If a separate flying machine were made instead, it would cause serious problems._

In addition, there are such generalized signal extensions, where the signal is not just a Redstone signal.

![img.gif](img/2.2.2-信号拓展.gif)

For ease of observation, I also made the main flying machine into a semi-dynamic engine. When the main flying machine moves, the semi-dynamic engine as an extension detects the movement signal and follows along.
