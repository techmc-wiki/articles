# #21 Mobility

---

In this section, our goal is to _extend "mobility" into structures_ and then _make those structures mobile_. In other words, instead of mounting things onto engines, we'll make functional structures fly directly.

## Basic Mobility Structures

In the previous section, we explained the principles of mobility and provided the basic 10gt flying machine. From the 10gt flying machine, we can extract the following mobility structure:

![img.png](img/2.1.1-10gt.png)

The necessary structures are:

- Two opposing pistons, at least one being a sticky piston
- Two observers, each activating one piston
- Slime blocks for connection

To extend a functional structure into a flying machine, simply fill in these three necessary components.

Since functional structures almost always contain slime blocks (though occasionally they don't), prioritize using existing slime blocks. It's best not to introduce new ones.

Next, if the functional structure contains observer-piston structures, prioritize using the existing observers. That is, add pistons to positions where observers can activate them. It's best not to introduce new observers.

Finally, place the pistons. Be careful to avoid exceeding the push limit and ensure all structures can be connected and moved.

## Examples

What follows is practice. This section can't really be broken down further—it's just practice makes perfect. Also, the principles above aren't set in stone. Some players prefer introducing new slime blocks or observers, which can actually make the whole machine cleaner and more elegant.

In practice, TNT duplicators mostly use ready-made structures, such as the bomber module from ilmango's world eater in the previous section.

### c) Other Structures

Structures vary wildly—there's really no unified standard to measure them by. So we won't go into detail from here on. Just try to satisfy the three points above.

Similarly, the various priorities aren't mandatory either. Heck, the engine doesn't even have to be 10gt. We just need to satisfy the principles of `1.2.1`.
